# ACL 2026 submission draft

Working draft of the paper:

> **Search, Fetch, or Answer: Commercial Search APIs Reshape the Retrieval Budget of LLM Agents Without Changing Their Accuracy**

## Files

```text
main.tex                # 8-page long paper body + appendix
refs.bib                # Bibliography (placeholders for proposal reference)
acl.sty                 # ACL style file (committed from acl-org/acl-style-files)
acl_natbib.bst          # ACL natbib bibliography style
acl_template/           # Full upstream ACL style files (formatting.md, samples)
Makefile                # `make` to build, `make clean` to tidy
```

## Build

Requires TeX Live (or MacTeX) with `pdflatex` and `bibtex`:

```bash
cd paper
make            # full build (pdflatex → bibtex → pdflatex × 2)
make quick      # single-pass build for fast iteration
make clean      # remove aux files
```

## Toggling submission modes

The first non-comment line of `main.tex` controls visibility:

```latex
\usepackage[review]{acl}    # blind submission + line numbers (default)
\usepackage[preprint]{acl}  # non-anonymous + page numbers (for arXiv)
\usepackage[final]{acl}     # camera-ready
```

## Validation

```bash
cd figures
python3 make_figures.py        # regenerates 3 data figures + validates against paper
python3 make_architecture.py   # regenerates the architecture diagram
```

`make_figures.py` prints `ALL OK — every figure number matches the paper.`
when all hardcoded numbers in the paper match what is computed live from
the JSONL trace and judge files. Any drift is reported as a `MISMATCH:`
line.
